# 08 - 开工指南

## 📌 文档说明
本文档为前端优先开发策略的开工指南,适用于FastAPI学习期间先行开发前端的场景。提供详细的开发顺序、Mock数据方案、以及如何利用现有设计文档进行全面编码。

## 1. 开发背景与策略

### 1.1 当前情况
- ✅ **设计文档**: 需求、架构、前后端设计、数据库、接口规范已完成
- ✅ **前端技能**: 3年Vue开发经验,React语法已学完
- ⏳ **后端学习**: FastAPI学习中,预计1-2个月完成
- 🎯 **开发策略**: 前端先行,使用Mock数据,后端跟进

### 1.2 开发顺序(推荐)

你的想法**非常合理**!前端先行的开发顺序如下:

```
阶段1: 基础搭建 (1周)
  ├─ 仓库初始化
  ├─ 前端三端项目脚手架
  └─ Mock服务搭建

阶段2: 用户端开发 (4-6周)
  ├─ 基础框架 (1周)
  ├─ 商品浏览 (2周)
  ├─ 购物车+订单 (2周)
  └─ 用户中心 (1周)

阶段3: 商家端开发 (4-5周)
  ├─ 基础框架 (1周)
  ├─ 商品管理 (2周)
  ├─ 订单管理 (1周)
  └─ 数据统计 (1周)

阶段4: 管理端开发 (3-4周)
  ├─ 基础框架 (1周)
  ├─ 审核管理 (1周)
  ├─ 数据监控 (1周)
  └─ 系统管理 (1周)

阶段5: 后端开发 (FastAPI学习完成后)
  ├─ 基础框架 (1周)
  ├─ 用户模块 (1周)
  ├─ 商品模块 (1周)
  ├─ 订单模块 (1周)
  └─ 其他模块 (2-3周)

阶段6: 前后端联调 (2周)
  └─ 替换Mock,对接真实API
```

### 1.3 为什么这个顺序合理?

**优势**:
1. **前端先行**: 利用Vue经验快速上手React,保持开发节奏
2. **业务理解**: 通过前端开发深入理解业务流程和数据结构
3. **接口设计验证**: Mock过程中验证接口设计的合理性
4. **后端参考**: 前端完成后,后端开发有明确的接口需求参考
5. **学习缓冲**: FastAPI学习期间不耽误项目进度

**注意事项**:
1. **严格遵循接口设计**: Mock数据必须与接口文档完全一致
2. **数据结构一致**: 前端数据模型要与数据库设计对应
3. **业务逻辑对齐**: 前端状态流转要与后端业务设计一致

## 3. Mock服务搭建

### 3.1 技术选型

推荐使用 **json-server** + **自定义中间件**:
- **json-server**: 快速搭建RESTful API
- **优势**: 零配置、自动生成CRUD、支持分页排序
- **扩展**: 可添加中间件实现JWT验证、复杂业务逻辑

### 3.2 Mock服务搭建步骤

#### Step 1: 初始化Mock项目
```bash
cd mock-server
npm init -y
npm install json-server cors express
```

#### Step 2: 创建Mock数据文件
```javascript
// mock-server/db.json
{
  "users": [],
  "merchants": [],
  "products": [],
  "categories": [],
  "orders": [],
  "addresses": [],
  "carts": [],
  "banners": []
}
```

#### Step 3: 创建自定义服务器
```javascript
// mock-server/server.js
const jsonServer = require('json-server');
const server = jsonServer.create();
const router = jsonServer.router('db.json');
const middlewares = jsonServer.defaults();

// 自定义路由
server.use(middlewares);
server.use(jsonServer.bodyParser);

// JWT验证中间件(简化版)
server.use((req, res, next) => {
  if (req.path === '/api/v1/users/login' || req.path === '/api/v1/users/register') {
    next();
  } else if (req.path.startsWith('/api/v1/users/me')) {
    const token = req.headers.authorization;
    if (!token) {
      return res.status(401).json({ code: 401, message: '未登录' });
    }
    next();
  } else {
    next();
  }
});

// 自定义登录接口
server.post('/api/v1/users/login', (req, res) => {
  const { username, password } = req.body;
  const db = router.db;
  const user = db.get('users').find({ username }).value();
  
  if (user && user.password === password) {
    res.json({
      code: 200,
      message: 'success',
      data: {
        access_token: 'mock-token-' + user.id,
        token_type: 'bearer',
        user: {
          id: user.id,
          username: user.username,
          phone: user.phone,
          email: user.email,
          avatar: user.avatar,
          nickname: user.nickname
        }
      }
    });
  } else {
    res.status(400).json({
      code: 400,
      message: '用户名或密码错误'
    });
  }
});

// 包装响应格式
router.render = (req, res) => {
  const data = res.locals.data;
  res.json({
    code: 200,
    message: 'success',
    data: data
  });
};

server.use('/api/v1', router);
server.listen(3000, () => {
  console.log('Mock Server is running on http://localhost:3000');
});
```

#### Step 4: 启动Mock服务
```bash
node server.js
```

### 4.2 基础代码结构

#### API封装
```typescript
// src/api/request.ts
import axios from 'axios';

const request = axios.create({
  baseURL: '/api/v1',
  timeout: 10000,
});

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    const { code, message, data } = response.data;
    if (code === 200 || code === 201) {
      return data;
    } else {
      return Promise.reject(new Error(message));
    }
  },
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default request;
```

```typescript
// src/stores/useAuthStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import type { User } from '@/types/user';

interface AuthState {
  user: User | null;
  token: string | null;
  setAuth: (user: User, token: string) => void;
  logout: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      user: null,
      token: null,
      setAuth: (user, token) => {
        localStorage.setItem('token', token);
        set({ user, token });
      },
      logout: () => {
        localStorage.removeItem('token');
        set({ user: null, token: null });
      },
    }),
    {
      name: 'auth-storage',
    }
  )
);
```

## 7. Mock数据与后端设计对齐

### 7.1 数据结构对齐原则

**关键**: Mock数据必须与数据库设计完全一致!

```typescript
// 错误示例 ❌
interface Product {
  id: number;
  title: string;  // 数据库字段是name,不是title!
}

// 正确示例 ✅
interface Product {
  id: number;
  name: string;  // 与数据库设计一致
  merchant_id: number;
  category_id: number;
  description?: string;
  images: string[];
  video?: string;
  status: string;
  sales: number;
  created_at: string;
  skus: ProductSKU[];
}
```

### 7.2 业务逻辑对齐

**订单状态流转** (参考后端设计):
```typescript
enum OrderStatus {
  PENDING_PAYMENT = 'pending_payment',    // 待付款
  PENDING_SHIPMENT = 'pending_shipment',  // 待发货
  SHIPPED = 'shipped',                    // 待收货
  COMPLETED = 'completed',                // 已完成
  CANCELLED = 'cancelled',                // 已取消
  REFUNDING = 'refunding',                // 退款中
  REFUNDED = 'refunded',                  // 已退款

// 前端状态流转逻辑要与后端一致
const canPay = status === OrderStatus.PENDING_PAYMENT;
const canCancel = status === OrderStatus.PENDING_PAYMENT;
const canConfirm = status === OrderStatus.SHIPPED;
```

## 10. 常见问题FAQ

### Q1: Mock数据如何模拟分页?
json-server自动支持分页:
```
GET /api/v1/products?_page=1&_limit=20
```

### Q2: 如何Mock文件上传?
使用 **multer** 中间件:
```javascript
const multer = require('multer');
const upload = multer({ dest: 'uploads/' });

server.post('/api/v1/upload', upload.single('file'), (req, res) => {
  res.json({
    code: 200,
    message: 'success',
    data: {
      url: `http://localhost:3000/uploads/${req.file.filename}`
    }
  });
});
```

### Q3: 前端三端有很多重复代码怎么办?
创建 `frontend/shared` 目录,抽取公共组件、工具函数、类型定义:
```typescript
// frontend/shared/components/ProductCard.tsx
// 三端都可以使用的商品卡片组件

// frontend/shared/utils/format.ts
// 三端都可以使用的格式化工具

// frontend/shared/types/product.ts
// 三端都可以使用的商品类型定义
```

### Q4: 如何保证Mock数据与后端一致?
1. 严格按照数据库设计定义数据结构
2. 使用TypeScript类型约束
3. 定期Review接口文档
4. 后端开发时参考前端已实现的功能

### Q5: 前端开发完成后,后端还没学完怎么办?
1. 继续优化前端(性能、体验、动画)
2. 补充单元测试
3. 完善文档
4. 学习后端相关知识
5. 准备部署方案
