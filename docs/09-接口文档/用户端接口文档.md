# 用户端接口文档

## 📌 文档说明
本文档详细定义用户端(C端)所有API接口,包括请求参数、响应格式、业务规则、Mock数据示例。前端开发时严格按照本文档实现。

---

## 1. 基础信息

### 1.1 接口基础URL
```
开发环境(Mock): http://localhost:3000/api/v1
生产环境: http://your-domain.com/api/v1
```

### 1.2 通用请求头
```http
Content-Type: application/json
Authorization: Bearer {token}  # 需要认证的接口
```

### 1.3 通用响应格式
```typescript
interface Response<T> {
  code: number;        // 200成功, 400参数错误, 401未认证, 403无权限, 404不存在, 500服务器错误
  message: string;     // 提示信息
  data: T | null;      // 响应数据
}
```

## 2. 用户认证模块

### 2.1 用户注册

**接口**: `POST /users/register`

**权限**: 公开

**请求参数**:
```typescript
interface RegisterRequest {
  username: string;    // 用户名, 3-50字符
  phone: string;       // 手机号, 11位
  email?: string;      // 邮箱(可选)
  password: string;    // 密码, 6-20字符
  code: string;        // 验证码, 4-6位
}
```

**请求示例**:
```json
{
  "username": "zhangsan",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "password": "123456",
  "code": "1234"
}
```

**响应数据**:
```typescript
interface User {
  id: number;
  username: string;
  phone: string;
  email?: string;
  avatar?: string;
  nickname?: string;
  status: string;      // normal, disabled
  created_at: string;
}
```

**响应示例**:
```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "phone": "13800138000",
    "email": "zhangsan@example.com",
    "avatar": null,
    "nickname": "zhangsan",
    "status": "normal",
    "created_at": "2024-02-24T10:00:00Z"
  }
}
```

**业务规则**:
- 用户名唯一
- 手机号唯一
- 验证码有效期5分钟
- 注册成功后自动登录

**权限**: 公开

**响应示例**:
```json

  "code": 200,
  "message": "登录成功",
  "data": {

      "username": "zhangsan",
      "phone": "13800138000",
```

**业务规则**:

## 5. 订单模块

### 5.1 创建订单

**接口**: `POST /orders`

**权限**: 需要登录

**请求参数**:
```typescript
interface CreateOrderRequest {
  items: OrderItemInput[];  // 订单商品
  address_id: number;       // 收货地址ID
  remark?: string;          // 用户备注
}

interface OrderItemInput {
  sku_id: number;
  quantity: number;
}
```

**请求示例**:
```json
{
  "items": [
    {
      "sku_id": 1,
      "quantity": 2
```

**响应数据**:
```typescript
interface Order {

  user_id: number;
  merchant_id: number;
  receiver_name: string;
  receiver_phone: string;
  receiver_address: string;
  total_amount: number;      // 商品总价
  freight: number;           // 运费
  discount_amount: number;   // 优惠金额
  actual_amount: number;     // 实付金额
  status: string;            // 订单状态
  logistics_company?: string;
  logistics_no?: string;
  remark?: string;
  merchant_remark?: string;
  paid_at?: string;
  shipped_at?: string;
  completed_at?: string;
  created_at: string;
  items: OrderItem[];
}

interface OrderItem {
  id: number;
  order_id: number;
  product_id: number;
  sku_id: number;
  product_name: string;
  sku_name?: string;
  image: string;
  price: number;
  quantity: number;
  subtotal: number;
}
```

## 8. 订单状态说明

```typescript
enum OrderStatus {
  PENDING_PAYMENT = 'pending_payment',    // 待付款
  PENDING_SHIPMENT = 'pending_shipment',  // 待发货
  SHIPPED = 'shipped',                    // 待收货
  COMPLETED = 'completed',                // 已完成
  CANCELLED = 'cancelled',                // 已取消
  REFUNDING = 'refunding',                // 退款中
  REFUNDED = 'refunded',                  // 已退款
}
```

**状态流转**:
```
pending_payment → (支付) → pending_shipment
                ↓ (取消)
              cancelled

pending_shipment → (发货) → shipped
                 ↓ (退款)
               refunding → refunded

shipped → (确认收货) → completed
        ↓ (退货退款)
      refunding → refunded
```
