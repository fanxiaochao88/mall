# 06 - 接口设计规范

## 📌 文档说明
本文档定义RESTful API设计规范、接口命名规则、请求响应格式,确保接口设计的一致性和规范性。

---

## 1. RESTful API设计原则

### 1.1 基本原则
- **资源导向**: URL表示资源,HTTP方法表示操作
- **无状态**: 每个请求包含完整信息,服务器不保存客户端状态
- **统一接口**: 统一的URL结构、请求格式、响应格式
- **分层系统**: 客户端无需知道是否直接连接到服务器

### 1.2 HTTP方法

| 方法 | 说明 | 示例 |
|-----|------|------|
| GET | 获取资源 | GET /api/v1/products |
| POST | 创建资源 | POST /api/v1/products |
| PUT | 完整更新资源 | PUT /api/v1/products/1 |
| PATCH | 部分更新资源 | PATCH /api/v1/products/1 |
| DELETE | 删除资源 | DELETE /api/v1/products/1 |

---

## 2. URL设计规范

### 2.1 URL结构
```
https://api.example.com/api/v1/resources/{id}/sub-resources
```

### 2.2 命名规则
- **小写字母**: 全部使用小写字母
- **复数名词**: 资源使用复数形式,如 `/users`, `/products`
- **连字符**: 多个单词使用连字符,如 `/order-items`
- **版本号**: 使用 `/api/v1/` 前缀

### 2.3 URL示例

| 操作 | 方法 | URL |
|-----|------|-----|
| 获取用户列表 | GET | /api/v1/users |
| 获取用户详情 | GET | /api/v1/users/{id} |
| 创建用户 | POST | /api/v1/users |
| 更新用户 | PUT | /api/v1/users/{id} |
| 删除用户 | DELETE | /api/v1/users/{id} |
| 获取用户订单 | GET | /api/v1/users/{id}/orders |
| 获取商品列表 | GET | /api/v1/products |
| 搜索商品 | GET | /api/v1/products?keyword=手机 |

---

## 3. 请求格式

### 3.1 请求头

```http
Content-Type: application/json
Authorization: Bearer {token}
Accept: application/json
```

### 3.2 请求参数

#### 路径参数
```
GET /api/v1/products/{id}
```

#### 查询参数
```
GET /api/v1/products?category_id=1&page=1&page_size=20
```

#### 请求体 (JSON)
```json
POST /api/v1/products
{
  "name": "iPhone 15",
  "category_id": 1,
  "price": 5999,
  "stock": 100
}
```

### 3.3 分页参数

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| page | int | 1 | 页码 |
| page_size | int | 20 | 每页数量 |

---

## 4. 响应格式

### 4.1 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 4.2 成功响应

#### 单个资源
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "iPhone 15",
    "price": 5999
  }
}
```

#### 列表资源
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "page": 1,
    "page_size": 20,
    "items": [
      {"id": 1, "name": "iPhone 15"},
      {"id": 2, "name": "iPad Pro"}
    ]
  }
}
```

#### 创建成功
```json
{
  "code": 201,
  "message": "创建成功",
  "data": {
    "id": 1,
    "name": "iPhone 15"
  }
}
```

### 4.3 错误响应

```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

## 5. HTTP状态码

| 状态码 | 说明 | 使用场景 |
|-------|------|---------|
| 200 | OK | 请求成功 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 删除成功 |
| 400 | Bad Request | 参数错误 |
| 401 | Unauthorized | 未认证 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突 |
| 422 | Unprocessable Entity | 验证失败 |
| 500 | Internal Server Error | 服务器错误 |

## 7. 接口文档示例

### 7.1 用户注册

**接口**: POST /api/v1/users/register

**请求参数**:
```json
{
  "username": "zhangsan",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "password": "123456",
  "code": "1234"
}
```

**响应示例**:
```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "zhangsan",
    "phone": "13800138000",
    "email": "zhangsan@example.com",
    "created_at": "2024-02-24T10:00:00"
  }
}
```

### 7.2 获取商品列表

**接口**: GET /api/v1/products

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| category_id | int | 否 | 分类ID |
| keyword | string | 否 | 搜索关键词 |
| min_price | decimal | 否 | 最低价格 |
| max_price | decimal | 否 | 最高价格 |
| sort_by | string | 否 | 排序:default,price_asc,price_desc,sales,new |
| page | int | 否 | 页码,默认1 |
| page_size | int | 否 | 每页数量,默认20 |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "page": 1,
    "page_size": 20,
    "items": [
      {
        "id": 1,
        "name": "iPhone 15",
        "price": 5999,
        "image": "https://example.com/image.jpg",
        "sales": 100
      }
    ]
  }
}
```
