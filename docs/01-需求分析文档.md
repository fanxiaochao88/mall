# 01 - 需求分析文档

## 📌 文档说明
本文档详细描述企业级商城项目的业务需求、功能需求、非功能需求,以及未来迭代规划。

---

## 1. 项目背景

### 1.1 业务场景
构建一个支持**自营+第三方商家入驻**的综合性电商平台,类似于京东、天猫的模式:
- **平台方**: 提供基础设施、流量、技术支持
- **商家**: 入驻平台,销售商品,独立运营
- **用户**: 在平台浏览、购买商品

### 1.2 核心价值
- **对用户**: 丰富的商品选择、便捷的购物体验、可靠的售后保障
- **对商家**: 低成本开店、获取流量、完善的管理工具
- **对平台**: 佣金收入、广告收入、数据价值

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| **页面加载时间** | < 2秒 | 首屏加载时间 |

| 需求 | 说明 |

## 5. 业务流程

### 5.1 用户购物流程

```
用户注册/登录
    ↓
浏览商品(首页/分类/搜索)
    ↓
查看商品详情
    ↓
加入购物车(或直接购买)
    ↓
确认订单(选择地址、优惠券)
    ↓
提交订单
    ↓
支付订单
    ↓
等待发货
    ↓
确认收货
    ↓
评价商品(可选)
```

### 5.2 商家入驻流程

```
商家注册
    ↓
填写店铺信息
    ↓
上传资质证明
    ↓
提交审核
    ↓
平台审核(通过/拒绝)
    ↓
开通店铺
    ↓
发布商品
```

### 5.3 订单状态流转

```
待付款 → (用户支付) → 待发货
                    ↓
                (商家发货)
                    ↓
                 待收货
                    ↓
            (用户确认收货/自动确认)
                    ↓
                 已完成
                    ↓
                (用户评价)
                    ↓
                 已评价

特殊流程:
待付款 → (超时/用户取消) → 已取消
待发货 → (用户申请退款) → 退款中 → 已退款
待收货 → (用户申请退货) → 退货中 → 已退款
```

### 5.4 售后流程

```
用户申请售后
    ↓
商家审核
    ↓
同意 → 用户退货(如需) → 商家确认收货 → 退款
    ↓
拒绝 → 用户可申诉 → 平台介入
```

## 6. 数据字典(核心实体)

### 6.1 用户(User)
- user_id: 用户ID
- username: 用户名
- phone: 手机号
- email: 邮箱
- password: 密码(加密)
- avatar: 头像
- status: 状态(正常/禁用)
- created_at: 注册时间

### 6.2 商家(Merchant)
- merchant_id: 商家ID
- shop_name: 店铺名称
- shop_logo: 店铺logo
- contact_name: 联系人
- contact_phone: 联系电话
- status: 状态(待审核/正常/禁用)
- created_at: 入驻时间

### 6.3 商品(Product)
- product_id: 商品ID
- merchant_id: 所属商家
- category_id: 分类ID
- product_name: 商品名称
- price: 价格
- stock: 库存
- images: 商品图片
- status: 状态(上架/下架)
- created_at: 创建时间

### 6.4 订单(Order)
- order_id: 订单ID
- user_id: 用户ID
- merchant_id: 商家ID
- total_amount: 订单金额
- status: 订单状态
- created_at: 下单时间
- paid_at: 支付时间

### 6.5 订单商品(OrderItem)
- item_id: 商品项ID
- order_id: 订单ID
- product_id: 商品ID
- quantity: 数量
- price: 单价
