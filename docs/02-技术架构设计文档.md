## 3. 核心技术方案

### 3.1 用户认证方案

#### JWT Token认证流程

```
1. 用户登录
   ↓
2. 后端验证用户名密码
   ↓
3. 生成JWT Token(包含user_id、角色等)
   ↓
4. 返回Token给前端
   ↓
5. 前端存储Token(localStorage)
   ↓
6. 后续请求携带Token(Authorization: Bearer <token>)
   ↓
7. 后端验证Token有效性
   ↓
8. 返回数据
```

#### Token设计

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 7200,
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Token内容**:
```json
{
  "sub": "user_id",
  "role": "user/merchant/admin",
  "exp": 1234567890
}
```

**安全措施**:
- ✅ Access Token有效期: 2小时
- ✅ Refresh Token有效期: 7天
- ✅ Token存储: localStorage(XSS风险需防范)
- ✅ HTTPS传输
- ✅ 敏感操作二次验证

### 3.2 权限控制方案

#### RBAC(基于角色的访问控制)

```
用户(User) ──→ 角色(Role) ──→ 权限(Permission)
```

**角色定义**:
- **普通用户**: 浏览商品、下单、查看订单
- **商家**: 管理商品、处理订单、查看数据
- **管理员**: 审核商家、管理商品、查看所有数据
- **超级管理员**: 系统配置、管理员管理

**实现方式**:
```python
# 后端装饰器
@require_role("admin")
async def delete_user(user_id: int):
    pass

# 前端路由守卫
const ProtectedRoute = ({ role }) => {
  const user = useUserStore();
  if (!user.hasRole(role)) {
    return <Navigate to="/403" />;
  }
  return <Outlet />;
};
```

#### 缓存更新策略

```python
# 1. Cache Aside(旁路缓存)
def get_product(product_id):
    # 先查缓存
    cache_data = redis.get(f"product:{product_id}")
    if cache_data:
        return cache_data
    
    # 缓存未命中,查数据库
    db_data = db.query(Product).filter_by(id=product_id).first()
    
    # 写入缓存
    redis.setex(f"product:{product_id}", 1800, db_data)
    return db_data

# 2. 更新时删除缓存
def update_product(product_id, data):
    db.query(Product).filter_by(id=product_id).update(data)
    redis.delete(f"product:{product_id}")
```

```python
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 加密
hashed_password = pwd_context.hash("user_password")

# 验证
pwd_context.verify("user_password", hashed_password)
```
