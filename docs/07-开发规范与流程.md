## 1. 代码规范

### 1.1 前端代码规范

#### TypeScript/JavaScript规范
- **使用TypeScript**: 所有代码使用TypeScript编写
- **命名规范**:
  - 变量/函数: 小驼峰 `userName`, `getUserInfo()`
  - 类/组件: 大驼峰 `UserCard`, `ProductList`
  - 常量: 全大写下划线 `API_BASE_URL`
  - 文件名: 小驼峰或短横线 `userCard.tsx`, `user-card.tsx`

#### React组件规范
```tsx
// ✅ 推荐
interface UserCardProps {
  user: User;
  onEdit: (id: number) => void;
}

export const UserCard: React.FC<UserCardProps> = ({ user, onEdit }) => {
  return (
    <div className="user-card">
      <h3>{user.name}</h3>
      <button onClick={() => onEdit(user.id)}>编辑</button>
    </div>
  );
};

// ❌ 不推荐
export default function UserCard(props: any) {
  return <div>{props.user.name}</div>;
}
```

#### 代码组织
```
src/
├── components/       # 通用组件
│   ├── Button/
│   │   ├── index.tsx
│   │   ├── Button.tsx
│   │   └── Button.module.css
├── pages/           # 页面组件
├── hooks/           # 自定义Hooks
├── utils/           # 工具函数
├── types/           # TypeScript类型
└── api/             # API接口
```

### 1.2 后端代码规范

#### Python规范 (PEP 8)
- **命名规范**:
  - 变量/函数: 小写下划线 `user_name`, `get_user_info()`
  - 类: 大驼峰 `UserService`, `ProductModel`
  - 常量: 全大写下划线 `MAX_PAGE_SIZE`
  - 文件名: 小写下划线 `user_service.py`

#### FastAPI路由规范
```python
# ✅ 推荐
from fastapi import APIRouter, Depends
from app.schemas.user import UserCreate, UserResponse
from app.services.user_service import user_service

router = APIRouter()

@router.post("/", response_model=UserResponse, status_code=201)
def create_user(
    user_in: UserCreate,
    db: Session = Depends(get_db)
) -> UserResponse:
    """创建用户"""
    user = user_service.create_user(db, user_in)
    return user

# ❌ 不推荐
@router.post("/create")
def create(data):
    return {"user": data}
```

## 2. Git工作流

### 2.1 分支策略 (Git Flow)

```
main (生产环境)
  ↑
develop (开发环境)
  ↑
feature/* (功能分支)
hotfix/* (紧急修复)
release/* (发布分支)
```

### 2.2 分支命名规范

| 分支类型 | 命名格式 | 示例 |
|---------|---------|------|
| 主分支 | main | main |
| 开发分支 | develop | develop |
| 功能分支 | feature/功能名 | feature/user-login |
| 修复分支 | bugfix/问题描述 | bugfix/fix-cart-bug |
| 紧急修复 | hotfix/问题描述 | hotfix/fix-payment |
| 发布分支 | release/版本号 | release/v1.0.0 |

### 2.3 工作流程

#### 开发新功能
```bash
# 1. 从develop创建功能分支
git checkout develop
git pull origin develop
git checkout -b feature/user-login

# 2. 开发功能,提交代码
git add .
git commit -m "feat: 实现用户登录功能"

# 3. 推送到远程
git push origin feature/user-login

# 4. 创建Pull Request到develop
# 5. Code Review通过后合并
# 6. 删除功能分支
git branch -d feature/user-login
```

#### 紧急修复
```bash
# 1. 从main创建hotfix分支
git checkout main
git checkout -b hotfix/fix-payment

# 2. 修复bug,提交代码
git commit -m "fix: 修复支付回调bug"

# 3. 合并到main和develop
git checkout main
git merge hotfix/fix-payment
git checkout develop
git merge hotfix/fix-payment

# 4. 删除hotfix分支
git branch -d hotfix/fix-payment
```

## 3. Commit规范

### 3.1 Commit Message格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 3.2 Type类型

| Type | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | feat: 添加用户登录功能 |
| fix | 修复bug | fix: 修复购物车数量计算错误 |
| docs | 文档 | docs: 更新README |
| style | 格式 | style: 格式化代码 |
| refactor | 重构 | refactor: 重构订单模块 |
| test | 测试 | test: 添加用户模块单元测试 |
| chore | 构建 | chore: 更新依赖版本 |

### 3.3 示例

```bash
# 好的commit
git commit -m "feat(user): 实现用户注册功能"
git commit -m "fix(cart): 修复购物车数量不更新的bug"
git commit -m "docs: 更新API文档"

# 不好的commit
git commit -m "update"
git commit -m "fix bug"
git commit -m "修改"
```

## 4. 代码审查 (Code Review)

### 4.1 审查清单

#### 功能性
- ✅ 功能是否符合需求
- ✅ 是否有边界情况处理
- ✅ 错误处理是否完善

#### 代码质量
- ✅ 代码是否易读易懂
- ✅ 是否有重复代码
- ✅ 命名是否规范
- ✅ 是否有注释(复杂逻辑)

#### 性能
- ✅ 是否有性能问题
- ✅ 数据库查询是否优化
- ✅ 是否有N+1查询

#### 安全
- ✅ 是否有SQL注入风险
- ✅ 是否有XSS风险
- ✅ 敏感数据是否加密

### 4.2 审查流程
1. 开发者提交Pull Request
2. 自动运行CI/CD(测试、Lint)
3. 指定审查人员
4. 审查人员Review代码
5. 提出修改意见
6. 开发者修改
7. 审查通过,合并代码

## 5. 测试规范

### 5.1 测试分类

| 测试类型 | 说明 | 工具 |
|---------|------|------|
| 单元测试 | 测试单个函数/组件 | Jest, Pytest |
| 集成测试 | 测试模块间交互 | Jest, Pytest |
| E2E测试 | 测试完整流程 | Playwright |

### 5.2 前端测试

```tsx
// 组件测试
import { render, screen, fireEvent } from '@testing-library/react';
import { UserCard } from './UserCard';

describe('UserCard', () => {
  it('should render user name', () => {
    const user = { id: 1, name: 'John' };
    render(<UserCard user={user} />);
    expect(screen.getByText('John')).toBeInTheDocument();
  });

  it('should call onEdit when button clicked', () => {
    const onEdit = jest.fn();
    const user = { id: 1, name: 'John' };
    render(<UserCard user={user} onEdit={onEdit} />);
    
    fireEvent.click(screen.getByText('编辑'));
    expect(onEdit).toHaveBeenCalledWith(1);
  });
});
```

### 5.3 后端测试

```python
# 单元测试
import pytest
from app.services.user_service import user_service

def test_create_user(db):

        username="test",

    user = user_service.create_user(db, user_in)
    assert user.username == "test"
    assert user.phone == "13800138000"

# API测试
```
