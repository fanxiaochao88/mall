# 商品列表页开发文档

## 一、页面概述

商品列表页是一个通用的商品展示页面，支持多种入口和参数组合，用于展示符合条件的商品列表。

### 1.1 页面入口

| 入口位置 | 携带参数 | 说明 |
|---------|---------|------|
| 首页搜索框 | `keyword` | 用户输入关键词搜索 |
| 推荐模块"更多"按钮 | `type` | 展示特定类型商品（recommend/new/hot） |
| 楼层模块"更多"按钮 | `category_id` | 展示特定分类商品 |

### 1.2 技术栈

- React + TypeScript
- React Router (路由和URL参数)
- TanStack Query (数据获取，已封装 `useProducts`)
- Ant Design (UI组件)
- Tailwind CSS (样式)

### 2.2 URL参数设计

使用 URL Search Params 传递所有筛选条件，便于分享和书签：

```
/products?keyword=手机&category_id=11&sort_by=price_asc&page=2
```

**支持的URL参数**（对应 `ProductListQuery` 接口）：

| 参数 | 类型 | 说明 | 示例 |
|-----|------|------|------|
| `type` | string | 商品类型 | `recommend`, `new`, `hot`, `all` |
| `category_id` | number | 分类ID | `11` |
| `keyword` | string | 搜索关键词 | `手机` |
| `min_price` | number | 最低价格 | `1000` |
| `max_price` | number | 最高价格 | `5000` |
| `sort_by` | string | 排序方式 | `default`, `price_asc`, `price_desc`, `sales`, `new` |
| `page` | number | 页码 | `1` |
| `page_size` | number | 每页数量 | `20` |

## 五、状态管理和数据流

### 5.1 URL 作为单一数据源

所有筛选条件存储在 URL 中，组件从 URL 读取参数：

```typescript
// 使用 useSearchParams 读取和更新 URL
const [searchParams, setSearchParams] = useSearchParams()

// 读取参数
const keyword = searchParams.get('keyword') || ''
const categoryId = searchParams.get('category_id')
  ? Number(searchParams.get('category_id'))
  : undefined

// 更新参数
const updateParams = (newParams: Partial<ProductListQuery>) => {
  const params = new URLSearchParams(searchParams)
  Object.entries(newParams).forEach(([key, value]) => {
    if (value !== undefined && value !== null && value !== '') {
      params.set(key, String(value))
    } else {
      params.delete(key)
    }
  })
  setSearchParams(params)
}
```

### 5.2 数据流向

```
URL参数 → useProducts Hook → React Query → API请求 → 数据展示
   ↑                                                      ↓
   └──────────────── 用户操作（筛选/排序/分页）──────────┘
```

### 5.3 useProducts Hook 使用

## 七、组装主页面

### Step 5: 在主组件中组装所有子组件

**文件**: `src/pages/products/ProductListPage.tsx`

**完整代码**:
```typescript
import { useSearchParams } from 'react-router-dom'
import { useProducts } from '@/hooks/serverData/useProducts'
import { Pagination } from 'antd'
import type { ProductListQuery } from '@/types'
import SearchBar from './components/SearchBar'
import FilterBar from './components/FilterBar'
import ProductList from './components/ProductList'

export default function ProductListPage() {
  const [searchParams, setSearchParams] = useSearchParams()

  const initialParams: ProductListQuery = {
    type: searchParams.get('type') as any,
    category_id: searchParams.get('category_id') ? Number(searchParams.get('category_id')) : undefined,
    keyword: searchParams.get('keyword') || undefined,
    sort_by: (searchParams.get('sort_by') as any) || 'default',
    page: searchParams.get('page') ? Number(searchParams.get('page')) : 1,
    page_size: 20,
  }

  const { products, total, isLoading, error, params, setParams } = useProducts(initialParams)

  const updateParams = (newParams: Partial<ProductListQuery>) => {
    const urlParams = new URLSearchParams(searchParams)
    Object.entries(newParams).forEach(([key, value]) => {
      if (value !== undefined && value !== null && value !== '') {
        urlParams.set(key, String(value))
      } else {
        urlParams.delete(key)
      }
    })
    setSearchParams(urlParams)
    setParams(prev => ({ ...prev, ...newParams }))
  }

  const handleSearch = (keyword: string) => {
    updateParams({ keyword, page: 1 })
  }

  const handleSortChange = (sort_by: string) => {
    updateParams({ sort_by, page: 1 })
  }

  const handleReset = () => {
    // 保留入口参数（type 和 category_id）
    const type = searchParams.get('type')
    const category_id = searchParams.get('category_id')

    // 构建重置后的参数（保留入口参数，清除筛选参数）
    const resetParams: ProductListQuery = {
      type: type as any,
      category_id: category_id ? Number(category_id) : undefined,
      page: 1,
      page_size: 20,
      sort_by: 'default',
    }

    // 更新 URL（只保留入口参数）
    const params = new URLSearchParams()
    if (type) params.set('type', type)
    if (category_id) params.set('category_id', category_id)

    setSearchParams(params)
    setParams(resetParams)
  }

  const handlePageChange = (page: number) => {
    updateParams({ page })
    window.scrollTo({ top: 0, behavior: 'smooth' })
  }

  return (
    <div className="container mx-auto px-4 py-6">
      <SearchBar defaultValue={params.keyword} onSearch={handleSearch} />
      <FilterBar sortBy={params.sort_by} onSortChange={handleSortChange} onReset={handleReset} />
      <ProductList products={products} isLoading={isLoading} error={error} />

      {!isLoading && products.length > 0 && (
        <div className="flex justify-center mt-8">
          <Pagination
            current={params.page}
            total={total}
            pageSize={params.page_size}
            onChange={handlePageChange}
            showSizeChanger={false}
            showTotal={(total) => `共 ${total} 件商品`}
          />
        </div>
      )}
    </div>
  )
}
```

## 八、注意事项

### 8.1 URL 参数同步

**关键点**:
- URL 是唯一的数据源，所有筛选条件都存储在 URL 中
- 每次参数变化时，同时更新 URL 和 `useProducts` 的 params
- 用户刷新页面或分享链接时，能够保持当前的筛选状态

### 8.2 分页重置

**规则**:
- 搜索、排序、筛选等操作时，必须重置页码为 1
- 只有点击分页器时才改变页码

```typescript
// ✅ 正确：搜索时重置页码
const handleSearch = (keyword: string) => {
  updateParams({ keyword, page: 1 })
}

// ❌ 错误：不重置页码
const handleSearch = (keyword: string) => {
  updateParams({ keyword })
}
```

### 8.3 重置逻辑

**重要**：重置按钮应该保留入口参数，只清除用户的筛选操作。

**参数分类**：
- **入口参数**（保留）：`type`、`category_id` - 决定用户想看什么类型的商品
- **筛选参数**（清除）：`keyword`、`sort_by`、`min_price`、`max_price` - 用户的二次筛选

```typescript
// ✅ 正确：保留入口参数
const handleReset = () => {
  const type = searchParams.get('type')
  const category_id = searchParams.get('category_id')

  const resetParams: ProductListQuery = {
    type: type as any,
    category_id: category_id ? Number(category_id) : undefined,
    page: 1,
    page_size: 20,
    sort_by: 'default',
  }

  const params = new URLSearchParams()
  if (type) params.set('type', type)
  if (category_id) params.set('category_id', category_id)

  setSearchParams(params)
  setParams(resetParams)
}

// ❌ 错误：清除所有参数（包括入口参数）
const handleReset = () => {
  setSearchParams()
  setParams({ page: 1, page_size: 20, sort_by: 'default' })
}
```

**用户体验对比**：
```
✅ 正确流程：
用户从"热门推荐"进入 → /products?type=hot
用户搜索"手机" → /products?type=hot&keyword=手机
用户点击"重置" → /products?type=hot （保留了 type，只清除了 keyword）

❌ 错误流程：
用户从"热门推荐"进入 → /products?type=hot
用户搜索"手机" → /products?type=hot&keyword=手机
用户点击"重置" → /products （type 丢失了！）
```

### 8.4 性能优化

1. **React Query 自动缓存**: `useProducts` 使用 React Query，相同参数的请求会自动缓存
2. **骨架屏**: 使用 `ProductCardSkeleton` 提升加载体验
3. **分页滚动**: 切换页码时自动滚动到顶部

### 8.5 可选功能（后期扩展）

如果需要更复杂的筛选，可以添加：
- 价格区间筛选（min_price, max_price）
- 分类筛选下拉框（使用 `productApi.getCategories()`）
- 筛选标签展示（显示当前生效的筛选条件）
