# 商城客户端模块开发指导手册

## 项目概述

这是一个基于 React + TypeScript + Vite 的电商客户端应用，作为个人名片项目的一部分。整体架构包括：
- **客户端**（当前项目）：用户购物端
- **商户端**：商家管理端（待开发）
- **管理员端**：平台管理端（待开发）
- **后端**：FastAPI 实现（待开发）

## 已完成模块

### 1. 首页模块 ✅
- 轮播图展示（含骨架屏）
- 商品推荐模块
- 商品楼层模块
- 使用 react-query 进行数据管理

### 2. 商品列表页 ✅
**路径**: `src/pages/products/ProductListPage.tsx`

**核心功能**:
- URL 参数联动（type, category_id, keyword, sort_by, page）
- 筛选排序（综合、价格升序/降序、销量、最新）
- 分页展示（每页 20 条）
- 骨架屏加载状态
- 错误处理

**技术要点**:
- 使用 `useSearchParams` 管理 URL 参数
- 使用 `useProducts` hook 封装数据获取逻辑
- react-query 自动缓存和状态管理

## 技术规范

### 1. 目录结构
```
src/
├── pages/              # 页面组件
│   ├── products/       # 商品相关页面
│   ├── cart/           # 购物车页面
│   ├── orders/         # 订单相关页面
│   ├── checkout/       # 订单确认页面
│   ├── payment/        # 支付页面
│   └── profile/        # 个人中心页面
├── components/         # 公共组件
├── hooks/              # 自定义 Hooks
│   └── serverData/     # 数据获取 Hooks
├── api/                # API 接口
├── types/              # TypeScript 类型定义
└── utils/              # 工具函数
```

### 2. 命名规范
- **页面组件**: `XxxPage.tsx`（如 `ProductDetailPage.tsx`）
- **子组件**: 使用 PascalCase（如 `SKUSelector.tsx`）
- **Hooks**: 使用 `use` 前缀（如 `useProductDetail.ts`）
- **API 文件**: 使用小驼峰（如 `product.ts`）
- **类型文件**: 与功能模块对应（如 `product.ts`）

### 3. 数据获取模式
使用 react-query 统一管理服务端状态：

```typescript
// 示例：useProductDetail hook
import { useQuery } from '@tanstack/react-query'
import { productApi } from '@/api/product'

export function useProductDetail(id: number) {
  const { data, isLoading, error } = useQuery({
    queryKey: ['product', id],
    queryFn: () => productApi.getProductDetail(id),
    enabled: !!id, // 只有 id 存在时才请求
  })

  return {
    product: data,
    isLoading,
    error,
  }
}
```

### 4. 状态管理
- **服务端状态**: 使用 react-query
- **客户端状态**: 使用 useState/useReducer
- **URL 状态**: 使用 useSearchParams
- **全局状态**: 考虑使用 Context API 或 Zustand（如用户信息、购物车数量）

### 5. 样式规范
- 使用 Tailwind CSS 进行样式开发
- 使用 Ant Design 组件库
- 保持响应式设计（移动端优先）

### 6. 错误处理
- 使用统一的错误处理组件 `ErrorComponent`
- API 错误通过 react-query 的 error 状态处理
- 表单验证使用 react-hook-form

### 7. 加载状态
- 使用骨架屏提升用户体验
- 按钮操作时显示 loading 状态
- 长列表使用虚拟滚动或分页

## 开发顺序建议

1. **商品详情页** - 连接商品列表和购物车的关键页面
2. **购物车页** - 核心购物流程
3. **收货地址管理** - 订单确认的前置功能
4. **订单确认页** - 完成购物流程
5. **支付页面** - 支付流程（可先做模拟支付）
6. **订单列表页** - 订单管理
7. **订单详情页** - 订单详情查看
8. **个人中心页** - 用户信息管理

## 注意事项

1. **认证鉴权**: 所有需要登录的页面需要添加路由守卫
2. **数据校验**: 前端表单验证 + 后端接口验证
3. **性能优化**:
   - 图片懒加载
   - 路由懒加载
   - 组件按需加载
4. **用户体验**:
   - 操作反馈（Toast 提示）
   - 加载状态
   - 空状态处理
   - 错误状态处理
5. **移动端适配**: 确保所有页面在移动端正常显示和操作
